webpack基础

打包工具的作用

当开发时有以下情况时，代码无法直接在浏览器中运行：

  • 使用es6相关语法，使用TS
  • 使用Less / Sass等css预处理器
  • 使用React / Vue等框架

打包工具（webpack）可以将以上情况的代码编译成能被浏览器识别运行的js、css等语法，除此之外打包工具（webpack）还能压缩代码、做兼容性处理、提升代码性能等。

下载webpack

npm i webpack webpack-cli -D

随后便可使用webpack命令进行打包，默认输出目录为./dist

npx webpack 入口(entry) --mode=模式（开发 / 生产）

npx webpack ./src/main.js --mode=production

核心概念

  • 入口(entry)：指示 Webpack 从哪个文件开始打包
  • 输出(output)：指示 Webpack 打包完的文件输出到哪里去，如何命名等
  • 加载器(loader)：webpack 本身只能处理 js、json 等资源，loader用于解析其他资源，loader可以直接使用无需引入
  • 插件(plugins)：扩展 webpack 的功能，插件需引入，一般为构造函数，需在plugins中实例化
  • 模式(mode)：主要有开发模式(development)和生产模式(production)，在开发模式下只会编译 JS 中的 ES Module 语法，在生产模式下还能压缩代码

基本配置文件

在项目根目录中创建webpack.config.js文件作为webpack配置文件，当目录下有该文件时，npx webpack会读取该文件进行项目打包。webpack 是基于 node.js 运行的，所以该配置文件采用 common.js 模块化规范，主要结构如下：

module.exports = {
    // 入口
    entry: "",  // 入口路径
    
    // 输出
    output: {
        // 文件输出路径
        path: "",  // 输出必须为绝对路径
        // 文件名
        filename: "",
    },
    
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /正则/,  // 正则匹配文件名，该加载器只处理匹配到的文件
                // loader只可指定一个加载器，而use是loader数组从而可指定多个加载器
                // 可以是loader字符串，也可是loader对象
                use: [loader数组] // 执行顺序：从下到上 / 从右到左
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置
    ],
    // 模式
    mode: "" // development / production
}
  • 配置文件的文件路径和文件名也可自定义，使用命令npx webpack --config 配置文件即可将自定义文件作为webpack配置文件
  • 配置文件的输入输出目录是命令执行的目录，故相对路径是相对于执行目录，在使用到绝对路径(如__dirname)时需维护路径

基本使用

  • 使用前需先下载所需依赖包、loader和plugins

准备工作

  • 可以在output中添加clean，以自动清空上次内容(配置了开发服务器时此项无意义)
  • 可以在加载器rules数组的元素中添加generator，以配置每项rule处理文件的输出路径和文件名
  • 在开发模式下，可下载webpack-dev-server开发服务器，并在配置文件中通过devServer配置开发服务器，通过npx webpack serve开启开发服务器，以实现代码修改时自动打包到内存中更新，此时不会输出到外存中显示
const path = require('path');

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出路径及文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true, // 在开发服务器中无意义
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.(ttf|woff2?)$/,
                type: "asset/resource",
                generator: {
                    // 匹配到的文件输出路径及文件名
                    // 将图片文件输出到 static/media 目录中
                    // 将图片文件命名 [hash:10][ext][query]
                    // [hash:10]: hash值取10位，用于文件名唯一
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: 'static/media/[hash:10][ext][query]',
                }
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置
    ],
    // 模式
    mode: "development",
    // 开发服务器
    devServer: {
        host: "localhost", // 启动服务器域名
        port: "3000", // 启动服务器端口号
        open: true, // 是否自动打开浏览器
    },
}

处理HTML资源

  • 下载html-webpack-plugin插件，在webpack配置文件的plugins中注册
  • 注册时可提供模板html文件
  • 打包时在打包目录生成index.html，结构与提供的模板html文件一致
  • 自动引入所需js资源，与其他插件的配合下也会自动引入其他资源
  • HTML资源在生产模式下会自动压缩
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true,
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
        ],
    },
    // 插件
    plugins: [
        // 插件配置
        // 以 public/index.html 为模板创建文件
        // 新的html文件有两个特点：1. 内容和源文件一致 2. 自动引入打包生成的js等资源
        new HtmlWebpackPlugin({
            template: path.resolve(__dirname, 'public/index.html'),
        }),
    ],
    // 模式
    mode: "development",
}

处理样式资源

加载样式资源

  • 需下载style-loader css-loader less-loader sass-loader stylus-loader
  • less-loader sass-loader stylus-loader负责将对应的预处理样式资源编译成css资源
  • css-loader负责将css资源编译成commonjs的模块到js中
  • style-loader负责将js中css通过创建style标签添加到html文件中生效
  • 注意loader顺序
const path = require('path');

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出路径及文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true, // 在开发服务器中无意义
    },
    
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.css$/,  // 正则匹配文件名，只检测匹配到的文件
                use: [
                    // 执行顺序：从下到上/从右到左（顺序不可乱）
                    'style-loader', // 将js中css通过创建style标签添加到html文件中生效
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                ],
            },
            {
                test: /\.less$/,
                use: [
                    'style-loader', // 将js中css通过创建style标签添加到html文件中生效
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'less-loader', // 将less资源编译成css资源
                ],
            },
            {
                test: /\.s[ac]ss$/,
                use: [
                    'style-loader', // 将js中css通过创建style标签添加到html文件中生效
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'sass-loader', // 将sass/scss资源编译成css资源
                ],
            },
            {
                test: /\.styl$/,
                use: [
                    'style-loader', // 将js中css通过创建style标签添加到html文件中生效
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'stylus-loader', // 将stylus资源编译成css资源
                ],
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置
    ],
    // 模式
    mode: "development"
}

提取css文件

  • 上述中是通过style-loader将js中css通过创建style标签添加到html文件中生效，所以是加载js时才会创建style标签添加到html中，此时页面会出现闪屏现象
  • 通过mini-css-extract-plugin插件，在webpack配置文件的plugins中注册，可提取css样式为单独文件到指定目录
  • 该插件中包含一个loader，用于替换style-loader，此时样式不会在加载js时创建style标签，而是通过html-webpack-plugin插件，通过link引入css文件
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true,
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.css$/,  // 正则匹配文件名，只检测匹配到的文件
                use: [
                    // 执行顺序：从下到上/从右到左（顺序不可乱）
                    MiniCssExtractPlugin.loader, // 通过link引入css文件
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                ],
            },
            {
                test: /\.less$/,
                use: [
                    MiniCssExtractPlugin.loader, // 通过link引入css文件
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'less-loader', // 将less资源编译成css资源
                ],
            },
            {
                test: /\.s[ac]ss$/,
                use: [
                    MiniCssExtractPlugin.loader, // 通过link引入css文件
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'sass-loader', // 将sass/scss资源编译成css资源
                ],
            },
            {
                test: /\.styl$/,
                use: [
                    MiniCssExtractPlugin.loader, // 通过link引入css文件
                    'css-loader',  // 将css资源编译成commonjs的模块到js中
                    'stylus-loader', // 将stylus资源编译成css资源
                ],
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置

        // 以 public/index.html 为模板创建文件
        // 新的html文件有两个特点：1. 内容和源文件一致 2. 自动引入打包生成的js等资源
        new HtmlWebpackPlugin({
            template: path.resolve(__dirname, 'public/index.html'),
        }),

        // 提取css成单独文件
        new MiniCssExtractPlugin({
            // 定义输出文件名和目录
            filename: 'static/css/main.css',
        }),
    ],
    // 模式
    mode: "development",
}

css兼容性处理

  • 根据预设值，webpack会将css样式转换成向后兼容的样式(如添加前缀)
  • 通过postcss、postcss-loader，配置css兼容性处理loader，一般情况下设置预设为postcss-preset-env，三者均需下载
  • postcss-loader位于css-loader之后，预处理器loader之前(loader执行顺序为从后到前)，意为在webpack编译css样式前进行兼容性处理，且兼容性处理的为css样式
  • 需要写css兼容性配置，可在package.json中的browserslist中进行配置，同时可对开发模式/生产模式单独配置
// webpack.config.js

const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');

const getStyleLoaders = (pre) => [
    // 执行顺序：从下到上/从右到左（顺序不可乱）
    MiniCssExtractPlugin.loader, // 通过link引入css文件
    'css-loader', // 将css资源编译成commonjs的模块到js中
    {
        // css兼容性处理
        loader: 'postcss-loader',
        options: {
            postcssOptions: {
                plugins: [
                    'postcss-preset-env', // 智能预设
                ]
            }
        }
    },
    pre // 预处理器loader, 将预处理资源编译成css资源
].filter(Boolean);

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true,
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.css$/,  // 正则匹配文件名，只检测匹配到的文件
                use: getStyleLoaders(),
            },
            {
                test: /\.less$/,
                use: getStyleLoaders('less-loader'),
            },
            {
                test: /\.s[ac]ss$/,
                use: getStyleLoaders('sass-loader'),
            },
            {
                test: /\.styl$/,
                use: getStyleLoaders('stylus-loader'),
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置

        // 以 public/index.html 为模板创建文件
        // 新的html文件有两个特点：1. 内容和源文件一致 2. 自动引入打包生成的js等资源
        new HtmlWebpackPlugin({
            template: path.resolve(__dirname, 'public/index.html'),
        }),

        // 提取css成单独文件
        new MiniCssExtractPlugin({
            // 定义输出文件名和目录
            filename: 'static/css/main.css',
        }),
    ],
    // 模式
    mode: "development",
}

// package.json
{
    //css兼容性配置
    "browserslist": [
        "last 2 version",
        "> 1%",
        "not dead"
    ]
}

css压缩

  • HTML和JS资源在生产模式模式下会自动压缩，css则需借助css-minimizer-webpack-plugin插件，在webpack配置文件的plugins中注册
  • 开发模式压缩时是将每个样式文件压缩成一行，并保留文件注释
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const CssMinimizerPlugin = require('css-minimizer-webpack-plugin');

const getStyleLoaders = (pre) => [

    'css-loader', // 将css资源编译成commonjs的模块到js中

        // css兼容性处理
        loader: 'postcss-loader',
        options: {
            postcssOptions: {
                plugins: [
                    'postcss-preset-env', // 智能预设
                ]
            }
        }
    },
    pre // 预处理器loader, 将预处理资源编译成css资源
].filter(Boolean);

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {

        // 入口文件输出文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true,
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.css$/,  // 正则匹配文件名，只检测匹配到的文件

            },
            {
                test: /\.less$/,
                use: getStyleLoaders('less-loader'),
            },
            {
                test: /\.s[ac]ss$/,
                use: getStyleLoaders('sass-loader'),
            },
            {
                test: /\.styl$/,
                use: getStyleLoaders('stylus-loader'),
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置

        // 以 public/index.html 为模板创建文件
        // 新的html文件有两个特点：1. 内容和源文件一致 2. 自动引入打包生成的js等资源
        new HtmlWebpackPlugin({

处理静态资源(字体图标、文本、图片、多媒体)

  • 在webpack4时需下载file-loader和url-loader进行处理，在webpack5中已内置
  • 在webpack5中通过type='asset'来表示加载静态资源模块，无需配置额外的loader
  • type取值有多种，常见的如下
    • asset/resource发送一个单独的文件并导出 URL。之前通过使用 file-loader 实现。多用于字体图标和多媒体
    • asset/inline 导出一个资源的 data URI。之前通过使用 url-loader 实现。多用于svg
    • asset 在导出一个 data URI 和发送一个单独的文件之间自动选择。之前通过使用 url-loader，并且配置资源体积限制实现。多用于图片，因为转为data URI时可减少请求数量，但会让体积变大，故图片小时导出data URI，图片大时发送一个单独文件
    • asset/source 导出资源的源代码。之前通过使用 raw-loader 实现。多用于文本，导出文件内容
const path = require('path');

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, 'dist'),  // 绝对路径
        // 入口文件输出文件名
        filename: "main.js",
        // 自动清空上次打包的内容
        // 先清空path目录，再进行打包
        clean: true,
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.(ttf|woff2?|mp3|mp4|avi|mkv)$/,
                type: "asset/resource",
                generator: {
                    // 匹配到的文件输出路径
                    // 将文件输出到 static/media 目录中
                    // 将图片文件命名 [hash:10][ext][query]
                    // [hash:10]: hash值取10位
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: 'static/media/[hash:10][ext][query]',
                }
            },
            {
                test: /\.svg$/,
                type: "asset/inline",
            },
            {
                test: /\.(png|jpe?g|gif|bmp)$/,
                type: "asset",
                parser: {
                    dataUrlCondition: {
                        maxSize: 10 * 1024 // 小于10kb的图片会被base64处理,即data URI
                    }
                },
                generator: {
                    // 匹配到的文件输出路径
                    // 将文件输出到 static/images 目录中
                    // 将图片文件命名 [hash:10][ext][query]
                    // [hash:10]: hash值取10位
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: 'static/images/[hash:10][ext][query]',
                },
            },
            {
                test: /\.txt$/,
                type: "asset/resource",
            },
        ],
    },
    // 插件
    plugins: [
        // 插件配置
    ],
    // 模式
    mode: "development",
}

处理JS资源

  • webpack默认只能处理ES Module语法，JS资源在生产模式下会自动压缩
  • 针对代码格式，我们使用 Eslint 来完成
  • 针对 js 兼容性处理，我们使用 Babel 来完成

JS代码格式检查(ESLint)

  • 下载eslint和eslint-webpack-plugin

  • 通过ESLint检查工具和eslint-webpack-plugin插件对JS代码进行格式检查，用于统一代码格式，便于团队开发

  • 配置文件位于项目根目录，命名有多种，如.eslintrc、.eslintrc.js和.eslintrc.json，或在package.json中配置eslintConfig，ESLint 会查找和自动读取它们

  • 常用的配置选项有extends 继承、parserOptions解析选项和rules规则

  • rules的每项规则有三种取值，rules中配置的规则会覆盖extends中继承的规则，rules规则目录详见ESLint文档

    • "off" 或 0 : 关闭规则
    • "warn" 或 1 : 开启规则，使用警告级别的错误：warn (不会导致程序退出)
    • "error" 或 2: 开启规则，使用错误级别的错误：error (当被触发的时候，程序会退出)

JS兼容性处理(Babel)

  • 下载 @babel/core、和babel-loader，以及对应的预设

  • 通过Babel(JS编译器)和babel-loader，将编写的JS代码(主要是ES6语法)转换为向后兼容的 JavaScript 语法，以便能够运行在当前和旧版本的浏览器或其他环境中

  • 配置文件位于项目根目录，命名有多种，如babel.config.js、babel.config.json、.babelrc、.babelrc.js和.babelrc.json，或在package.json中配置babel，Babel会查找和自动读取它们

  • 主要配置选项是presets 预设，简单理解为一组 Babel 插件, 扩展 Babel 功能

    • @babel/preset-env: 一个智能预设，允许您使用最新的 JavaScript。
    • @babel/preset-react：一个用来编译 React jsx 语法的预设
    • @babel/preset-typescript：一个用来编译 TypeScript 语法的预设

  • 在新版本中箭头函数默认不再转换为普通函数，可以在output中禁用箭头函数

开发模式示例

  • ./config/webpack.dev.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const ESLintWebpackPlugin = require('eslint-webpack-plugin');

const getStyleLoaders = (pre) => [
    // 执行顺序：从下到上/从右到左（顺序不可乱）
    MiniCssExtractPlugin.loader, // 通过link引入css文件
    'css-loader', // 将css资源编译成commonjs的模块到js中
    {
        // css兼容性处理
        loader: 'postcss-loader',
        options: {
            postcssOptions: {
                plugins: [
                    'postcss-preset-env', // 智能预设
                ]
            }
        }
    },
    pre // 预处理器loader, 将预处理资源编译成css资源
].filter(Boolean);

module.exports = {
    // 入口
    entry: "./src/main.js",  // 相对路径
    // 输出
    output: {
        // 所有文件输出路径
        path: path.resolve(__dirname, '../dist'),  // 绝对路径
        // 入口文件输出文件名
        filename: "main.js",
        // 禁止箭头函数
        environment: {
            arrowFunction: false,
        },
    },
    // 加载器
    module: {
        rules: [
            // 加载器配置
            {
                test: /\.css$/,  // 正则匹配文件名，只检测匹配到的文件
                use: getStyleLoaders(),
            },
            {
                test: /\.less$/,
                use: getStyleLoaders('less-loader'),
            },
            {
                test: /\.s[ac]ss$/,
                use: getStyleLoaders('sass-loader'),
            },
            {
                test: /\.styl$/,
                use: getStyleLoaders('stylus-loader'),
            },

            {
                test: /\.(ttf|woff2?|mp3|mp4|avi|mkv)$/,
                type: "asset/resource",
                generator: {
                    // 匹配到的文件输出路径
                    // 将文件输出到 static/media 目录中
                    // 将图片文件命名 [hash:10][ext][query]
                    // [hash:10]: hash值取10位
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: 'static/media/[hash:10][ext][query]',
                }
            },
            {
                test: /\.svg$/,
                type: "asset/inline",
            },
            {
                test: /\.(png|jpe?g|gif|bmp)$/,
                type: "asset",
                parser: {
                    dataUrlCondition: {
                        maxSize: 10 * 1024 // 小于10kb的图片会被base64处理,即data URI
                    }
                },
                generator: {
                    // 匹配到的文件输出路径
                    // 将文件输出到 static/images 目录中
                    // 将图片文件命名 [hash:10][ext][query]
                    // [hash:10]: hash值取10位
                    // [ext]: 使用之前的文件扩展名
                    // [query]: 添加之前的query参数
                    filename: 'static/images/[hash:10][ext][query]',
                },
            },
            {
                test: /\.txt$/,
                type: "asset/resource",
            },

            {
                test: /\.js$/,
                exclude: /node_modules/, // 排除node_modules代码不编译
                loader: "babel-loader",
            },
        ]
    },
    // 插件
    plugins: [
        // 插件配置

        // 以 public/index.html 为模板创建文件
        // 新的html文件有两个特点：1. 内容和源文件一致 2. 自动引入打包生成的js等资源
        new HtmlWebpackPlugin({
            template: path.resolve(__dirname, '../public/index.html'),
        }),

        // 提取css成单独文件
        new MiniCssExtractPlugin({
            // 定义输出文件名和目录
            filename: 'static/css/main.css',
        }),

        // 使用ESLint进行js代码格式检查
        new ESLintWebpackPlugin({
            // 指定检查文件的根目录
            context: path.resolve(__dirname, "../src"),
        }),
    ],
    // 模式
    mode: "development",
    // 开发服务器
    devServer: {
        host: "localhost", // 启动服务器域名
        port: "3000", // 启动服务器端口号
        open: true, // 是否自动打开浏览器
    },
}
